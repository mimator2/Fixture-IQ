================================================================================
DATA FIELDS IN EACH CSV
================================================================================

season:       Season name (e.g., "2022-2023")
team_name:    Team name (matches Premier League roster for that season)
player_id:    Unique player identifier from API-Football
player_name:  Player name
injury_type:  "Missing Fixture" (confirmed absence - only value in output)
reason:       Injury description (e.g., "Thigh Injury", "Suspension", "Illness")
fixture_date: Date injury started (YYYY-MM-DD format)
end_date:     Date player returned to availability (YYYY-MM-DD) - may be NULL
days_out:     Calculated days of confirmed injury (only for "Missing Fixture")

================================================================================
DATA SOURCE & METHODOLOGY
================================================================================

API Endpoint:   API-Football (api-sports.io)
Primary Source: /sidelined endpoint (complete historical injury records)
Fallback:       /injuries endpoint with "Questionable" status filtered out
Data Quality:   Only "Missing Fixture" type included (removes uncertain injuries)
Rate Limiting:  Exponential backoff strategy to prevent API blocking
Extraction Date: May 20, 2026

Key Features:
  ✓ Confirmed injuries only - "Questionable" status excluded
  ✓ Handles season-based team changes (relegation/promotion)
  ✓ Calculated days_out metric for injury duration analysis
  ✓ Both individual team files and combined season files
  ✓ Smart mode for efficient re-extraction (skips completed teams)
